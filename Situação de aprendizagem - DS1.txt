Atividade Prática: Configuração de Servidores DNS, DHCP e Web em Ambiente Virtualizado

Objetivo:
Nesta atividade, você deverá configurar um ambiente virtualizado com duas máquinas virtuais: uma com Windows Server e outra com Linux. O Windows Server será responsável pelos serviços de DNS e DHCP, enquanto a máquina Linux irá hospedar um serviço Web (NGINX). A comunicação entre as máquinas será feita por meio de uma rede interna.

Requisitos da Atividade
1. Máquina Virtual Windows Server
Sistema Operacional: Windows Server 2019 ou superior 

Memória RAM: 4 GB

Processador: 2 núcleos

Disco: 40 GB

Serviços a configurar:

Servidor DNS:

Criar uma zona de pesquisa direta (ex: senai.local)

Adicionar registros A e CNAME conforme necessário para apontar para a máquina Linux.

Servidor DHCP:

Configurar um escopo de IPs (ex: 192.168.100.10 a 192.168.100.50)

Garantir que a máquina Linux receba IP automaticamente.

2. Máquina Virtual Linux (Cliente e Servidor Web)
Distribuição sugerida: Ubuntu Server .

Memória RAM: 2 GB

Processador: 1 núcleo

Disco: 20 GB

Configuração de rede: DHCP (receber IP do Windows Server)

Serviço a configurar:

Servidor Web ( NGINX):

Instalar e configurar o serviço web

Criar uma página HTML simples exibindo informações como nome da máquina e IP

Testar o acesso via navegador  partir do Windows Server e Linux 

Certificar-se de que o nome do site esteja acessível via nome DNS configurado (ex: senai.local)

Ambiente de Rede
Configure ambas as máquinas na mesma rede interna (Internal Network) para que se comuniquem exclusivamente entre si.

Entrega:
Print das configurações das máquinas virtuais (memória, CPU, disco, rede)

Print ou vídeo demonstrando:

Atribuição de IP automático à VM Linux via DHCP

Resolução de nome do servidor web via DNS

Acesso à página web no servidor Linux e Windows
